### Модели
#### 1. User
#### 2. Привычка:
- Пользователь — создатель привычки.
- Место — место, в котором необходимо выполнять привычку.
- Время — время, когда необходимо выполнять привычку.
- Действие — действие, которое представляет собой привычка.
- Признак приятной привычки — привычка, которую можно привязать к выполнению полезной привычки.
- Связанная привычка — привычка, которая связана с другой привычкой, важно указывать для полезных привычек, но не для приятных.
- Периодичность (по умолчанию ежедневная) — периодичность выполнения привычки для напоминания в днях.
- Вознаграждение — чем пользователь должен себя вознаградить после выполнения.
- Время на выполнение — время, которое предположительно потратит пользователь на выполнение привычки.
- Признак публичности — привычки можно публиковать в общий доступ, чтобы другие пользователи могли брать в пример чужие привычки.

### Валидаторы
- Исключить одновременный выбор связанной привычки и указания вознаграждения.
- В модели не должно быть заполнено одновременно и поле вознаграждения, и поле связанной привычки. Можно заполнить только одно из двух полей.
- Время выполнения должно быть не больше 120 секунд.
- В связанные привычки могут попадать только привычки с признаком приятной привычки.
- У приятной привычки не может быть вознаграждения или связанной привычки.
- Нельзя выполнять привычку реже, чем 1 раз в 7 дней.
- Нельзя не выполнять привычку более 7 дней. Например, привычка может повторяться раз в неделю, но не раз в 2 недели. За одну неделю необходимо выполнить привычку хотя бы один раз.

## Запуск проекта:
### 1. Шаги по запуску проекта через docker-compose
#### Предварительные требования
- Перед запуском проекта убедитесь, что у вас установлены:
Docker;Docker Compose;файл .env с необходимыми переменными окружения 
- Клонируйте репозиторий:
```
git clone https://github.com/kirillgladkikh/course_5.git
```
- Создайте файл .env согласно шаблону [.env.sample](.env.sample).

#### Шаги по запуску проекта
- Проверьте конфигурацию Docker Compose (опционально, для выявления ошибок в синтаксисе):
```
docker-compose config
```
- Запустите все сервисы в фоновом режиме:
```
docker-compose up -d
```
- Дождитесь инициализации сервисов (может занять 1–2 минуты, особенно для БД).

### 2. Команда для запуска проекта (в фоновом режиме):
```
docker-compose up -d
```
Эта команда:
- соберёт образы (если нужно);
- запустит контейнеры для всех сервисов (db, redis, backend, celery, celery-beat);
- применит миграции БД через python manage.py migrate;
- запустит Django‑сервер и Celery‑воркеры.

### 3. Проверка работоспособность каждого из сервисов
#### База данных PostgreSQL (db)

Статус:
```
docker-compose ps db
```
Ожидаемый статус: Up (или Up (healthy), если healthcheck пройден).

Логи:
```
docker-compose logs db
```
Убедитесь, что нет ошибок инициализации.
Проверка подключения:
Подключитесь к контейнеру и проверьте БД:
```
docker-compose exec db psql -U ${DATABASE_USER} -d ${DATABASE_NAME}
```
Внутри psql выполните \l для списка БД.

#### Redis (redis)

Статус:
```
docker-compose ps redis
```
Ожидаемый статус: Up.

Логи:
```
docker-compose logs redis
```
Проверка ping:
Выполните внутри контейнера:
```
docker-compose exec redis redis-cli ping
```
Ожидаемый ответ: PONG.

#### Бэкенд Django (backend)

Статус:
```
docker-compose ps backend
```
Ожидаемый статус: Up (healthy).

Логи:
```
docker-compose logs backend
```
Ищите строки:

Operations to perform: ... и Running migrations... (применяются миграции);

Starting development server at http://0.0.0.0:8000/ (сервер запущен).

Проверка HTTP‑ответа:
Откройте в браузере: http://localhost:8000 или выполните:
```
curl http://localhost:8000/
```
Ожидаемый код ответа: 200 OK.

#### Celery Worker (celery)

Статус:
```
docker-compose ps celery
```
Ожидаемый статус: Up.

Логи:
```
docker-compose logs celery
```
Ищите строку: 

[INFO/MainProcess] Connected to redis://redis:6379// (успешное подключение к Redis).

#### Celery Beat (celery-beat)

Статус:
```
docker-compose ps celery-beat
```
Ожидаемый статус: Up.

Логи:
```
docker-compose logs celery-beat
```
Ищите строку: 
[INFO] beat: Starting... (планировщик запущен).
